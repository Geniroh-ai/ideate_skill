User Guide · v1.0

ideate

McKinsey-style pitch-deck wizard for Claude Code
github.com/Geniroh-ai/ideate_skill ↗

ideate turns raw context — emails, briefs, notes, source material — into versioned, McKinsey-aesthetic pitch decks through a guided conversational wizard. Every project gets a structured repo. Markdown is the source of truth. HTML is generated deterministically. Every slide is checked pixel-perfect for content overflow before you ship.

01

Install

Personal skill (single user)
Drop the skill into Claude Code's skills directory.
# clone the repo
git clone https://github.com/Geniroh-ai/ideate_skill.git /tmp/ideate
cp -R /tmp/ideate/skills/ideate ~/.claude/skills/
Plugin (shareable across team)
Install as a Claude Code plugin from the git remote.
# clone into Claude's plugins folder
git clone https://github.com/Geniroh-ai/ideate_skill.git \
  ~/.claude/plugins/ideate
Prefer SSH? Open the repo at github.com/Geniroh-ai/ideate_skill, click the green Code button, switch the dropdown to SSH, and copy that URL instead. Same install steps otherwise.
Requirements. Claude Code (CLI / VS Code / JetBrains / web). Node.js (for compile + overflow detector). Google Chrome / Chromium / Edge (for PDF export + pixel-perfect fit check).

03

What gets created on disk

# every deck project lives in its own folder
<deck-name>/
├── .ideate/                 # state, design choice, version count
├── context/                 # YOU drop source material here
├── assets/                  # images, screenshots, logos
├── markdown/
│   ├── current.md           # SOURCE OF TRUTH — edit this directly to iterate
│   ├── previous/            # versioned snapshots, timestamped
│   └── DELTA.md             # changelog: what changed across versions
├── html/                    # compiled, timestamped HTML versions
└── pdf/                     # exported PDFs, timestamped
Why markdown is the source of truth. The HTML is always derivable from the markdown. To iterate on content, edit markdown/current.md directly and ask Claude to "regenerate". You don't have to re-run the wizard.
04

Six design variations

DesignAestheticBest for
mckinsey-classicEditorial serif (Fraunces) on warm paper, navy + restrained red, action-titledStrategy decks, partnership proposals, board pitches
mckinsey-sleekAll-Inter sans, no italic serif, same warm paletteModern strategy decks, executive proposals
bcg-monochromeAlmost-black on cream, accent green, generous whitespaceDiagnostic / data-led decks, consulting reports
bain-boldStrong red verticals, charcoal headlines, large stat calloutsSales decks, investor updates, growth pitches
editorial-noirDark charcoal, ivory text, serif display, warm gold accentHigh-impact opening / closing decks
modern-minimalOff-white, charcoal, single cobalt accent, sans throughoutStartup pitches, product launches
Add your own. Drop a folder under designs/ with a design.json that defines tokens (palette, fonts, spacing). The skill picks it up on the next invocation.
05

Power-user shortcuts

Skip the full wizard for common cases. Each shortcut maps to a specific flow.

ShortcutWhat it does
/ideate quick "topic" "audience"Minimal Q&A, sensible defaults (mckinsey-classic, explore tone, 12 slides). For when you need a deck in two minutes.
/ideate from-md path/to/file.mdSkip the wizard entirely. Use existing markdown. Run compile.js directly.
/ideate regenerateInside a project — re-compile markdown/current.md to a new versioned HTML.
/ideate exportInside a project — export the latest HTML to PDF. Nothing else.
/ideate switch-design <name>Inside a project — change design and recompile. No content changes.
/ideate check-fit [<html>]Run pixel-perfect overflow detector against an HTML file (defaults to latest).
/ideate list-designsPrint the design catalogue.
/ideate helpPrint the shortcut table.
06

What the scripts do

compile.js
Deterministic markdown → HTML. Reads the structured markdown, picks the design's tokens, emits a single self-contained HTML file. Zero npm deps.
# usage
node compile.js \
  <markdown.md> \
  <design-name> \
  <out.html>
check-overflow.js
Pixel-perfect content fit check. Spawns headless Chrome via raw CDP (no puppeteer dep), measures every .slide for vertical/horizontal overflow + offending descendants. Exits 1 if anything overflows.
# usage
node check-overflow.js <in.html>
export-pdf.sh
Headless Chrome to PDF. 1280×720 page size, zero margins, no header/footer, background graphics on. Auto-detects Chrome / Chromium / Edge.
# usage
./export-pdf.sh <in.html> <out.pdf>
The fit check is gating. Step 11 of the wizard runs check-overflow.js after compile. If any slide overflows, Claude tightens the markdown automatically and recompiles. The deck never ships with overflowing content.
07

Tone presets

explore
Exploratory partnership / early conversation. Soft asks, "could", "might", "draft", "indicative". No hard commitments.
commit
Confident proposal. Concrete metrics, clear asks, definite verbs ("delivers", "drives", "lifts"). Used when the relationship is established.
formal
Board / regulator / committee. Third-person, evidence-led, conservative numbers. Every claim sourced. No flourishes.
08

Iterating on a deck

  • Edit markdown directly. Open markdown/current.md, change content, ask Claude to "regenerate" — or run node compile.js yourself.
  • Switch design without losing content. /ideate switch-design bain-bold recompiles the same markdown with the new design.
  • Versioning is automatic. Every recompile moves the previous markdown into markdown/previous/ with a timestamp. DELTA.md records what changed.
  • Re-run the wizard inside an existing project for a shorter menu — iterate content, switch design, regenerate, or export.
